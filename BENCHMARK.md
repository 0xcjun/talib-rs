## Optimization Techniques

| Technique | Indicators | Typical Gain |
|-----------|-----------|-------------|
| Single-pass O(n) sliding SMA+STDDEV | BBANDS | 2-3x |
| Inline multi-layer EMA cascade (no intermediate Vec) | TEMA, TRIX, DEMA, T3 | 2-3x |
| C TA-Lib EMA formulation: `k*(x-prev)+prev` | EMA, MACD, TEMA, TRIX, T3, DEMA | 10-20% |
| O(n) sliding sums | LINEARREG family, BETA, TSF | 1.5-2.5x |
| Two-pass extremum (cache-friendly 2-way) | AROON, MINMAX, MINMAXINDEX | +10-35% |
| Slice iterator rescan (bounds elision) | MAX, MIN, WILLR, MIDPOINT, AROON | 10-40% |
| Inline TR (no temp Vec) | ATR | +35% |
| `mul_add` hardware FMA | EMA, MACD, DEMA, TEMA, TRIX, KAMA | ~7% |
| `Vec::with_capacity + extend` (no COW) | MOM, ROC, TRANGE, price transforms | 30-70% |
| `collect()` replacing `vec![0.0;n]` (no calloc COW) | SQRT, math transforms | +28% |
| `copysign` branchless pattern output | CDLDOJI, 57 CDL patterns | 2-3x |
| Branch elimination + shift pattern | SAR, SAREXT | +15-25% |
| MACDEXT → MACD delegation (all-EMA fast path) | MACDEXT | 2x |
| LTO fat + codegen-units=1 | All cross-crate calls | 1.1-1.3x |

> All algorithms are O(n). Zero `unsafe` blocks in entire codebase.
